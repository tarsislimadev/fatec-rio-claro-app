# Swagger Plan: Back-end Endpoints (Auth + Profile)

## Objective
Define an OpenAPI/Swagger contract for the first app scope: login, register, profile read/update, and logout.  
This contract should be implemented by backend and consumed by mobile app with no breaking changes.

## OpenAPI baseline
- **Spec version:** OpenAPI 3.0.3 (or 3.1.0 if preferred by backend tooling)
- **Base URL (prod):** `https://api.fatecrc.com`
- **Base URL (dev):** `https://api-dev.fatecrc.com`
- **Auth scheme:** `Bearer JWT` (`Authorization: Bearer <token>`)
- **Content type:** `application/json`

## Endpoint plan
| Method | Path | Auth | Purpose |
|---|---|---|---|
| POST | `/auth/register` | No | Create account |
| POST | `/auth/login` | No | Authenticate user and issue token |
| POST | `/auth/logout` | Yes | Invalidate current session/token |
| GET | `/profile/me` | Yes | Get logged-in user profile |
| PUT | `/profile/me` | Yes | Update logged-in user profile |

## Request/response contracts

### 1) POST `/auth/register`
**Request body**
```json
{
  "name": "Ana Souza",
  "email": "ana@fatec.sp.gov.br",
  "password": "StrongPass123!",
  "passwordConfirmation": "StrongPass123!"
}
```

**Success (201)**
```json
{
  "user": {
    "id": "usr_123",
    "name": "Ana Souza",
    "email": "ana@fatec.sp.gov.br",
    "role": "student",
    "createdAt": "2026-04-13T13:00:00Z",
    "updatedAt": "2026-04-13T13:00:00Z"
  },
  "token": "jwt-token",
  "tokenType": "Bearer",
  "expiresIn": 3600
}
```

**Errors**
- `400` validation error
- `409` email already in use

### 2) POST `/auth/login`

### 3) POST `/auth/logout`
**Headers**
- `Authorization: Bearer <token>`

**Success (204)**
- Empty body

**Errors**
- `401` unauthorized/invalid token

## Shared schemas to define in Swagger components
- `UserAuthResponse` (user + token fields)
- `UserProfile`
- `RegisterRequest`
- `LoginRequest`
- `UpdateProfileRequest`
- `ApiError`
- `FieldError`

### Standard error shape
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Request validation failed",
    "fields": [
      { "field": "email", "message": "Invalid email format" }
    ],
    "requestId": "req_abc123"
  }
}
```

## Validation rules (v1)
- `name`: required, min 2, max 120
- `email`: required, valid format, normalized lowercase
- `password`: required, min 8, at least 1 uppercase, 1 lowercase, 1 number
- `passwordConfirmation`: must match `password`
- `phone`: optional, E.164-compatible string
- `course`: optional, max 50
- `avatarUrl`: optional, valid URL

## Security and lifecycle notes
- Access token TTL: 1 hour (configurable)
- Logout behavior:
  - If using stateless JWT only, logout is client-side token discard.
  - If using revocation list/refresh flow, backend invalidates token/session.
- Require HTTPS in all non-local environments.
- Include `requestId` in all errors for tracing.

## Swagger deliverables
1. Create `openapi.yaml` containing endpoints/schemas/security above.
2. Publish Swagger UI (e.g., `/docs`) and raw spec (e.g., `/openapi.json`).
3. Add contract tests to guarantee response shapes/status codes.
4. Version contract as `v1` and use non-breaking evolution rules.
